# 插件开发指南

插件系统统一了内建（builtin）和远程（remote）两种形态，都走同一套 manifest、Hook、Host API。

- **内建插件**：跟随后端启动，进程内执行
- **远程插件**：通过 WebSocket 接入，独立进程

## 环境

- Python 3.10+
- Node.js 22+

启动开发环境：

```powershell
.\start.bat restart              # Windows
bash ./start.sh restart          # Linux / WSL
python tools/start_launcher.py --tail-logs   # 前台日志
```

插件 WebSocket 地址：`ws://127.0.0.1:23331`

## 快速开始：远程插件

### 1. 创建项目

```bash
mkdir config/plugins/plugin-my-device && cd config/plugins/plugin-my-device && npm init -y
```

`package.json`：

```json
{
  "name": "@garlic-claw/plugin-my-device",
  "version": "0.1.0",
  "private": true,
  "main": "dist/index.js",
  "scripts": { "build": "tsc", "start": "node dist/index.js" },
  "dependencies": { "@garlic-claw/plugin-sdk": "*", "@garlic-claw/shared": "*" },
  "devDependencies": { "typescript": "^5.7.0" }
}
```

### 2. 写代码

```typescript
import { PluginClient } from '@garlic-claw/plugin-sdk/client';
import type { PluginCapability } from '@garlic-claw/shared';

const tools: PluginCapability[] = [
  { name: 'get_status', description: '获取设备状态', parameters: {} },
  { name: 'set_brightness', description: '设置亮度', parameters: { level: { type: 'number', description: '亮度 0-100', required: true } } },
];

const client = PluginClient.fromRemoteAccess({
  accessKey: process.env.PLUGIN_ACCESS_KEY ?? '',
  pluginName: 'my-device-living-room',
  remote: { auth: { mode: 'required' }, capabilityProfile: 'actuate', remoteEnvironment: 'iot' },
  serverUrl: process.env.WS_URL ?? 'ws://127.0.0.1:23331',
}, {
  manifest: {
    name: 'Living Room Device', version: '1.0.0', description: '客厅设备插件',
    permissions: [], tools, hooks: [],
    config: { type: 'object', items: { displayName: { type: 'string', description: '显示名称', defaultValue: '客厅设备' } } },
  },
});

client.onCommand('get_status', async () => ({ online: true, brightness: 75 }));
client.onCommand('set_brightness', async (params) => ({ success: true, brightness: params.level as number }));
client.connect();
```

SDK 子入口：

- `@garlic-claw/plugin-sdk/client` — `PluginClient`
- `@garlic-claw/plugin-sdk/host` — host facade 类型
- `@garlic-claw/plugin-sdk/authoring` — authoring helper

### 3. 配置远程接入

```bash
curl -X PUT http://127.0.0.1:23330/api/plugins/my-device-living-room/remote-access \
  -H "Authorization: Bearer <token>" -H "Content-Type: application/json" \
  -d '{"access":{"accessKey":"<key>","serverUrl":"ws://127.0.0.1:23331"},"displayName":"Living Room Device","version":"1.0.0","remote":{"remoteEnvironment":"iot","auth":{"mode":"required"},"capabilityProfile":"actuate"}}'
```

读取连接信息：

```bash
curl http://127.0.0.1:23330/api/plugins/my-device-living-room/remote-connection \
  -H "Authorization: Bearer <token>"
```

注意：
- `auth.mode=none` 不需要 accessKey
- `auth.mode=optional` 可选
- `auth.mode=required` 远端必须带和宿主一致的 accessKey
- `serverUrl` 和 `accessKey` 存在宿主这边，不要写进插件自己的 config

### 4. 构建和运行

```bash
npm run build && npm start
```

## Manifest

```typescript
interface PluginManifest {
  id: string;              // 由 pluginName 决定
  name: string;
  version: string;
  runtime: 'local' | 'remote';
  description?: string;
  permissions: PluginPermission[];
  tools: PluginCapability[];
  crons?: PluginCronDescriptor[];
  hooks?: PluginHookDescriptor[];
  config?: PluginConfigSchema;
  routes?: PluginRouteDescriptor[];
}
```

`hooks` 描述符：

```typescript
interface PluginHookDescriptor {
  name: PluginHookName;
  description?: string;
  priority?: number;   // 越小越早，同优先级按 ID 排序
  filter?: {
    message?: {
      commands?: string[];
      regex?: string | { pattern: string; flags?: string };
      messageKinds?: Array<'text' | 'image' | 'mixed'>;
    };
  };
}
```

`filter` 目前只对 `message:received` 生效。复杂过滤逻辑应在 handler 里自己判断。

`config` 里声明 `maxConcurrentExecutions`（数字类型）可以限制插件 `tool/route/hook` 的并发上限，默认 6。

## 权限

```text
conversation:read / conversation:write
config:read
automation:read / automation:write
cron:read / cron:write
llm:generate
log:read / log:write
kb:read
memory:read / memory:write
persona:read / persona:write
provider:read
storage:read / storage:write
subagent:run
state:read / state:write
user:read
```

调用 Host API 时会真正校验，不只是写在 manifest 里就完事了。插件还支持全局和会话级启停，聊天工具暴露、Hook 调度都会受这套规则影响。

## 工具

```typescript
client.onCommand('tool_name', async (params, context) => {
  // params: JsonObject
  // context: { callContext, host }
  return { result: '...' };
});
```

`callContext` 标明了谁在调用这个工具（聊天还是自动化等）。`host` 是统一 Host API 入口。

## Hook

### 完整列表

| Hook | 触发时机 |
|------|---------|
| `message:received` | 消息到达宿主 |
| `chat:before-model` | 模型调用前 |
| `chat:waiting-model` | 模型等待中 |
| `chat:after-model` | 模型返回后 |
| `conversation:created` | 新会话创建 |
| `conversation:history-rewrite` | 会话历史重写 |
| `message:created` | 消息入库 |
| `message:updated` | 消息修改 |
| `message:deleted` | 消息删除 |
| `automation:before-run` | 自动化执行前 |
| `automation:after-run` | 自动化执行后 |
| `tool:before-call` | 工具调用前 |
| `tool:after-call` | 工具调用后 |
| `response:before-send` | 回复发送前 |
| `response:after-send` | 回复发送后 |
| `subagent:before-run` | 子代理执行前 |
| `subagent:after-run` | 子代理执行后 |
| `plugin:loaded` | 插件加载 |
| `plugin:unloaded` | 插件卸载 |
| `plugin:error` | 插件出错 |
| `cron:tick` | 定时触发 |

### 注册

```typescript
client.onHook('chat:before-model', async (payload, context) => {
  return { action: 'mutate', systemPrompt: '追加的系统提示词' };
});
```

返回 `{ action: 'mutate', ... }` 改写请求，`{ action: 'pass' }` 跳过不做任何事，`{ action: 'short-circuit', assistantContent: '...' }` 跳过后续所有 Hook 直接返回。

### Hook payload 示例

`chat:before-model` 输入：

```typescript
{
  context: PluginCallContext,
  request: {
    providerId: string, modelId: string,
    systemPrompt: string,
    messages: PluginLlmMessage[],
    availableTools: PluginAvailableToolSummary[],
    variant?: string,
    providerOptions?: JsonObject,
  },
  mutation: { systemPrompt?: string; providerId?: string; modelId?: string; variant?: string; providerOptions?: JsonObject },
}
```

### commandGroup / sessionWaiter

SDK 在 `message:received` 之上封装了更高层的 API：

```typescript
client.command('help', () => '可用命令：/math add 1 2');
const math = client.commandGroup('math', { alias: ['m'], description: '数学命令' });
math.command('add', ({ args }) => `结果：${Number(args[0]) + Number(args[1])}`, { alias: ['sum'] });
client.onMessage((payload) => { /* 兜底处理 */ });
```

- `command(name, handler, { alias, priority })` — 注册命令
- `commandGroup(name, { alias, priority })` — 命令组，支持 `.command()` / `.group()` 嵌套
- `sessionWaiter(handler)` — 活动等待态命中时优先交给本地 waiter
- handler 可以返回字符串、`{ content, parts? }` 或原生 Hook 对象，SDK 会自动转换

## Route

插件可以声明 HTTP JSON Route：

```typescript
client.onRoute('my-route', async (request) => {
  return { status: 200, body: { ok: true } };
});
```

在 manifest 里声明 `routes: [{ name: 'my-route', method: 'POST', path: '/my-path' }]`。宿主会把匹配请求转发给插件。

## Cron

manifest 声明静态 cron：

```json
{ "crons": [{ "name": "heartbeat", "schedule": "*/5 * * * *" }] }
```

动态 cron 通过 Host API 注册：

```typescript
const job = await context.host.createCron({ name: 'cleanup', schedule: '0 * * * *' });
const jobs = await context.host.listCrons();
```

`DELETE /api/plugins/:pluginId/crons/:jobId` 只能删 `source=host` 的动态 cron。manifest 里声明的 cron 只能通过更新插件来修改。

## Host API

通过 `context.host` 调用，常用方法：

| 方法 | 说明 |
|------|------|
| `getConfig()` | 读取插件配置 |
| `getStorage(key)` / `setStorage(key, value)` / `deleteStorage(key)` | 插件私有 KV |
| `getState(key)` / `setState(key, value)` | 临时状态 |
| `saveMemory({ content, category?, keywords? })` | 写入记忆 |
| `searchMemories(query, limit?)` | 搜索记忆 |
| `generateText({ providerId, modelId, messages, ... })` | 调用 LLM |
| `createAutomation({ name, trigger, actions })` | 创建自动化 |
| `listAutomations()` | 列出自动化 |
| `createCron({ name, schedule })` / `listCrons()` | 动态 cron |
| `writeLog({ level, message })` / `listLogs(query?)` | 事件日志 |
| `getCurrentProvider()` / `listProviders()` | provider 信息 |
| `getConversation()` / `listConversationMessages()` | 当前会话 |

## 配置管理 API

宿主治理接口（管理员权限）：

```text
GET    /api/plugins/:pluginId/config
PUT    /api/plugins/:pluginId/config
GET    /api/plugins/:pluginId/llm-preference
PUT    /api/plugins/:pluginId/llm-preference
GET    /api/plugins/:pluginId/remote-connection
PUT    /api/plugins/:pluginId/remote-access
GET    /api/plugins/:pluginId/storage
PUT    /api/plugins/:pluginId/storage
DELETE /api/plugins/:pluginId/storage?key=...
GET    /api/plugins/:pluginId/scopes
PUT    /api/plugins/:pluginId/scopes
GET    /api/plugins/:pluginId/health
GET    /api/plugins/:pluginId/events
GET    /api/plugins/:pluginId/event-log
PUT    /api/plugins/:pluginId/event-log
GET    /api/plugins/:pluginId/crons
DELETE /api/plugins/:pluginId/crons/:jobId
GET    /api/plugins/:pluginId/sessions
DELETE /api/plugins/:pluginId/sessions/:conversationId
POST   /api/plugins/:pluginId/actions/:action
```

事件日志接口支持的查询参数：`limit`、`level`、`type`、`keyword`、`cursor`。

`PUT /api/plugins/:pluginId/remote-access` 在线创建或刷新远程插件信息。`GET /api/plugins/:pluginId/remote-connection` 返回 SDK 可以直接用的连接参数。
