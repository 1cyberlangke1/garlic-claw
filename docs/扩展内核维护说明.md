# 扩展内核维护说明

面向维护者。说明当前 owner 边界和做减法时要守住的线。

## 目标

把后端收敛成稳定内核：core 只保留宿主边界、治理和最少运行时原语。作者侧继续兼容 plugin / MCP / skill。

先读契约本身：[`docs/扩展内核契约说明.md`](./扩展内核契约说明.md)

## Owner 边界

### RuntimeHostService

负责需要宿主协作的 Host API 分发：对外暴露给插件的 host call 入口、需要跨 service 协作的宿主行为、需要组合多个 owner 的宿主级流程。纯投影、纯 reader、纯校验不要塞回来。

### RuntimeHostPluginDispatchService

统一插件 dispatch：`tool/hook/route` 的 builtin/remote 调度、runtime plugin snapshot 列表。插件选择、hook 过滤、remote request 往返、builtin handler 调度都放这里，不绕回 RuntimeHostService。

### RuntimeHostPluginRuntimeService

插件自己的 runtime 资源面：`storage.*`、`state.*`、`config.*`、`cron.*`。不混回 dispatch owner 或治理 owner。

### RuntimeHostKnowledgeService / RuntimeHostUserContextService

宿主本地读取视图：provider/persona/kb/memory/user 等读取侧语义。只读或投影宿主现有数据就放这里。

### RuntimeHostSubagentRunnerService

子代理任务执行与写回：request 组装、tool set 选择、运行结果写回。子代理能力不塞回通用 runtime owner。

### RuntimePluginGovernanceService

治理 owner：health-check、reload、reconnect、supportedActions。不承担通用 dispatch。

### PluginPersistenceService

插件持久化与只追加事件日志：config/scopes、plugin event log。文档和控制器都按这个 owner 维护。

## 维护规则

### 不回到旧结构

禁止：泛化 helper/facade/compatibility 壳、"只是换名字"的薄转发 service、旧式 catch-all owner。

### 只保留明确 owner

- 纯 dispatch → `RuntimeHostPluginDispatchService`
- 纯 runtime 资源 → `RuntimeHostPluginRuntimeService`
- 纯治理 → `RuntimePluginGovernanceService`
- 纯持久化/事件日志 → `PluginPersistenceService`
- 纯读取视图 → 对应 reader owner

### 删代码时同步删文档

每次删旧路径或 owner，同批更新 `docs/**` 等文件。代码删了文档还在放行是不行的。

## 验证要求

改动完成后：

- 对应 owner 的定向 spec
- `npm run typecheck -w packages/server`
- `npm run test -w packages/server -- --runInBand`
- `npm run build -w packages/server`
- `npm run smoke:server`
