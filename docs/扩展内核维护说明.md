# 扩展内核维护说明

面向维护者。说明当前各模块的职责边界和做减法时要守住的线。

目标：把后端收敛成稳定内核。core 只保留宿主边界、治理和最少运行时。开发者侧继续兼容 plugin / MCP / skill。

先读：[`docs/扩展内核契约说明.md`](./扩展内核契约说明.md)

## 模块职责

### PluginHostService

需要宿主协作才能完成的 Host API 分发：对插件的 host call 入口、跨 service 的宿主行为、需要组合多个模块的宿主流程。只做读取映射、只读访问、参数校验的代码不要塞回来。

### PluginDispatchService

统一插件调度：`tool/hook/route` 的 builtin/remote 路由、运行时插件快照列表。插件选择、hook 过滤、远程请求、builtin handler 调度都放这里。

### PluginRuntimeService

插件自己的运行时资源：`storage.*`、`state.*`、`config.*`、`cron.*`。这些不混进调度模块或治理模块。

### KnowledgeReaderService / UserContextService

宿主读取视图：provider/persona/kb/memory/user 等查询。只是读取或汇总已有数据就放这里。

### SubagentRunnerService

子代理任务的执行和结果写回：请求组装、工具集选择、结果写回。子代理能力不塞回通用运行时模块。

### RuntimePluginGovernanceService

治理：健康检查、重载、重连、支持的动作列表。不做通用调度。

### PluginPersistenceService

插件持久化和追加日志：config/scopes、事件日志。

## 维护规则

### 不回到老架构

不要新建 helper/facade/compatibility 之类的薄壳，不要搞"换个名字"的转发 service，不要回到 catch-all 的大模块模式。

### 代码归属清晰

- 调度逻辑 → `PluginDispatchService`
- 运行时资源 → `PluginRuntimeService`
- 治理 → `RuntimePluginGovernanceService`
- 持久化/日志 → `PluginPersistenceService`
- 只读查询 → 对应的 reader

### 删代码同时删文档

删掉旧的路径或模块时，同批更新文档。代码删了文档还写着那是不行的。

## 验证

改动完成后跑一遍：

- 对应模块的单元测试
- `npm run typecheck -w packages/server`
- `npm run test -w packages/server -- --runInBand`
- `npm run build -w packages/server`
- 日常快速回归优先 `npm run smoke:server:core`
- 全量后端契约回归跑 `npm run smoke:server`
- 需要保留浏览器失败证据时跑 `npm run smoke:web-ui:artifacts`
