# 扩展内核契约

定义 Garlic Claw 内核的稳定边界。

核心原则很简单：内核越小越好，只保留必要的基础设施。SDK 提供开发工具，但不把上层便利性固化进内核。

## 1. 统一在哪里

统一发生在运行时层面，不在开发格式层面。

- **plugin** — 宿主原生扩展
  - `packages/shared/src/types/plugin.ts`
  - `packages/server/src/plugin/plugin.module.ts`
  - `packages/plugin-sdk/src/authoring/`、`client/`、`host/`
- **MCP** — 外部能力协议，通过适配器接入
  - `packages/server/src/execution/mcp/mcp.service.ts`
  - `packages/server/src/execution/tool/tool-registry.service.ts`
- **skill** — 兼容 skill 生态
  - `packages/server/src/execution/skill/skill-tool.service.ts`
  - `packages/server/src/execution/skill/skill-registry.service.ts`

每种扩展格式由各自生态决定，内核只看它们最终暴露出来的能力接口。

## 2. 内核只有两种操作

- **动作调用**（action call）：插件工具、MCP 工具、skill 工具、host API
- **事件订阅**（event subscription）：插件 hook 链

以下是边界入口，不是新的核心操作：
- `route` — HTTP 入口
- `cron` — 宿主调度
- `subagent` — 受控子任务，底层还是走 action/hook/permission

## 3. 默认私有

所有能力默认不暴露，只有在 manifest/配置中显式声明后才可见：

- 插件：manifest 里声明了 `tools/hooks/routes/crons/config` 才进入运行时
- MCP：配置启用且连接成功后，工具才进入统一工具层
- skill：模型调用 `skill` 工具后，内容才注入上下文

扩展之间禁止直接互调。内核内部可以做快速通道优化（进程内调用、延迟解析、本地快照），但这些是实现细节，不是开发者能依赖的语义。

## 4. Skill 的两面

skill 不是第三套厚运行时：

- **面向 AI**：提示词、策略、目录发现、延迟加载 → `SkillToolService`
- **面向运行时**：加载策略、资源清单、工具入口 → `SkillRegistryService` + `ToolRegistryService`

skill 触发的执行最终回到通用工具链。依赖解析走能力契约，不绑死某种扩展格式。

## 5. 状态原语

内核只保留四种通用状态，不给每种新功能单独建表：

| 类型 | 接口 | 作用域 |
|------|------|--------|
| 私有 KV | `storage.*` | 插件 / 会话 / 用户 |
| 临时状态 | `state.*` | 插件 / 会话 / 用户 |
| 导出快照 | `provider.*`、`persona.*`、`kb.*`、对话/用户/插件信息查询 | — |
| 追加日志 | `log.list`、`log.write`、`PluginPersistenceService` 事件方法 | — |

只有宿主业务真正需要的实体才留在 core schema 里。

## 6. Builtin 的定位

builtin 是参考实现，不是特权实现。它和 remote 用同一套 manifest、权限、hook、route、host API。

已经从 core 迁到 builtin 的：记忆、知识库上下文、对话标题、自动化、人设路由、子代理、治理/响应/工具/生命周期记录等。

新功能优先考虑做成插件/MCP/skill/tool。builtin 的意义是演示"怎么消费内核"，不是给内核开新口子的理由。

## 7. 维护规则

加新功能时按这个顺序判断：

1. 做成一个插件/MCP/skill 适配器行不行？
2. 在现有 action/event 上加接口行不行？
3. 用 storage/state/快照/日志表达状态行不行？
4. 都不行，再考虑进内核。

禁止：
- 搞第二套扩展运行时
- 让 skill 绑死某插件格式
- 让 MCP 绕过统一工具层直连聊天
- 给单个 builtin 开内核私有入口
- 把实现优化暴露成开发者语义
